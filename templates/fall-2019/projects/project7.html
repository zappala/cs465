Read about the Unix password file formats, [/etc/passwd](http://www.cyberciti.biz/faq/understanding-etcpasswd-file-format/) and [/etc/shadow](http://www.cyberciti.biz/faq/understanding-etcshadow-file/). In particular, make sure you understand where and how usernames, salts, and hashed passwords are stored. 1. Create a simple program that can generate dummy passwd and shadow files, given user input of a username and password (plaintext). The input can be in whatever format you would like (single entry from command-line, list of entries from a file, etc.). You can use the crypt() function in C to create a Unix password style MD5 hash (include crypt.h). You may use any value you wish, constant or otherwise, for your salts, so long as they adhere to the salt charset rules. Example [passwd](/static/files/passwd.txt) and [shadow](/static/files/shadow.txt) files are available. 1. Test the passwd and shadow files generated by your program to ensure that they are formatted properly. You will run into problems if they are not. The pwck command should help with this. 1. Download and install the correct version of John the Ripper for your operating system. 1. Using the [documentation](http://www.openwall.com/john/doc/) (I also found [this](https://linuxconfig.org/password-cracking-with-john-the-ripper-on-linux) to be a useful start for linux users), figure out how to use John the Ripper. In particular, figure out how to use the “unshadow” utility bundled with John the Ripper, how to set minimum and maximum lengths for cracking, and how to use a custom wordlist. 1. Use this [password strength meter](http://www.passwordmeter.com/). to create passwords at various strength levels. For each strength type, make a password that contains: * Only lowercase letters * Both lowercase and uppercase letters * Lower and uppercase letters, numeric digits, and punctuation 1. Using the program you wrote in step 2, create passwd and shadow files for each password strength type. 1. Use John the Ripper to try and crack the passwords you have created. Start with the weak passwords and work your way up until you find that you are unable to crack the passwords in a realistic time frame. Feel free to use any heuristic that John the Ripper allows, and experiment with other wordlists too. There are many free wordlists online. We host a [decent one](/static/files/cracklib.txt), but there are far more robust ones elsewhere. You may also experiment with running John the Ripper on multiple machines, with each instance operating on a subset of your passwords. As John the Ripper runs, you may hit any key to view some statistics. One of these is comparisons per second (c/s). Roughly keep track of the average number of comparisons per second that John the Ripper attains during your cracking. 1. As you crack increasingly secure passwords, it may be useful to use multiple machines to run simultaneous instances of John the Ripper. If you choose to do this on the CS lab machines, please do not use any more than 6 machines. 1. Additionally, be sure to run your John the Ripper processes in the background in a high “nice” mode. This will make sure that our cracking processes do not interfere with other students' activities on the lab machines. To run a command with a high nice mode, you can use > nice -n 15 [command] [args] where [command] is the name of your binary and [args] are the arguments passed to it. If you fail to use nice, the sysadmins will likely have to kill your process and will disable your account. ## Questions Thoughtfully answer the questions below. Submit your answers for passoff. 1.
